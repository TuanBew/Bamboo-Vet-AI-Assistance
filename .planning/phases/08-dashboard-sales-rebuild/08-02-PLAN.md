**4. Helper: getFilteredCustomerIds** — Central filter function:

```typescript
async function getFilteredCustomerIds(
  db: ReturnType<typeof createServiceClient>,
  filters: DashboardFilters
): Promise<{ customerIds: string[]; allCustomers: Array<{ id: string; customer_type: string; customer_name: string; latitude: number | null; longitude: number | null }> }> {
  let query = db.from('customers').select('id, customer_type, customer_name, latitude, longitude, supplier_id')
  if (filters.npp) query = query.eq('supplier_id', filters.npp)
  if (filters.kenh === 'le') query = query.in('customer_type', KENH_LE_TYPES)
  else if (filters.kenh === 'si') query = query.in('customer_type', KENH_SI_TYPES)
  const { data } = await query
  const customers = (data ?? []) as Array<{ id: string; customer_type: string; customer_name: string; latitude: number | null; longitude: number | null; supplier_id: string }>
  return { customerIds: customers.map(c => c.id), allCustomers: customers }
}
```

**5. Main getDashboardData function:**

The function should:
a. Parse month string to get year/month numbers and compute start/end dates: `startOfMonth = '${year}-${String(month).padStart(2,'0')}-01'`, `endOfMonth` = last day of month
b. Fetch filtered customer IDs using `getFilteredCustomerIds`
c. Fetch all products into a Map<product_id, { product_group, manufacturer, industry, product_name }> (for JS-side joins)
d. Fetch npp_list from `suppliers`: `db.from('suppliers').select('id, supplier_name').order('supplier_name')`
e. Fetch filter_options: distinct `product_group` and `manufacturer` from products table
f. Fetch customer_purchases for selected month filtered by customerIds (and optionally by nganhHang/thuongHieu via product lookup):
   - Get all purchases: `.from('customer_purchases').select('*').gte('purchase_date', startOfMonth).lte('purchase_date', endOfMonth)`
   - Filter by customerIds: `.in('customer_id', customerIds)` (if customerIds.length > 0; else return empty data)
   - Do JS-side filter for nganhHang/thuongHieu by looking up product_id in the products Map
g. Fetch purchase_orders + purchase_order_items for nhap_hang metrics:
   - Orders: `.from('purchase_orders').select('id, supplier_id, order_date, total_amount')`
   - If npp filter: `.eq('supplier_id', filters.npp)`
   - Items: `.from('purchase_order_items').select('order_id, product_id, quantity, promo_qty, unit_price, subtotal')`
   - JS-side join items to orders; JS-side filter by nganhHang/thuongHieu via products Map
h. Compute yearly_series: aggregate customer_purchases.total_value (ban_hang) and purchase_orders.total_amount (nhap_hang) by year. Hardcode years 2022-2026 with 0 for missing years.
i. Compute monthly_series: aggregate by year+month, then run `computeForecast` independently for ban_hang and nhap_hang (map to `query_count` field), merge results.
j. Compute daily_series for selected month: aggregate by day from customer_purchases (ban_hang) and purchase_order_items via order dates (nhap_hang).
k. Compute metrics_box: nhap_hang sum for month, ban_hang sum for month, customers_active (distinct customer_ids with purchases in month), customers_total (total customers for NPP), sku_sold (distinct product_ids in purchases), sku_total (total products count), nhan_vien (distributor_staff count for NPP).
l. Compute pie_nhap and pie_ban: group purchase_order_items subtotals and customer_purchases total_values by product's industry (nganh), product_group (nhom), manufacturer (thuong_hieu) using the products Map.
m. Compute kpi_row: tong_nhap and tong_ban for selected month, tong_nhap_prev_year and tong_ban_prev_year for same month previous year, sl_ban (sum customer_purchases.qty), sl_km (sum purchase_order_items.promo_qty for month), avg_per_order (tong_ban / number of distinct purchase transactions).
n. Compute staff_list: fetch distributor_staff for selected NPP; for each staff, filter customer_purchases by staff_id for selected month; compute total_sales, order_count (distinct purchase_date+customer_id combos), avg_per_order, customer_count (distinct customer_ids), days_over_1m (days where sum > 1,000,000), daily_sparkline (array of 28-31 daily totals), by_nhom and by_thuong_hieu breakdowns via products Map.
o. Compute customer_section: by_type_sales (sum total_value by customer_type from filtered purchases), by_type_count (count customers by type from allCustomers), map_pins (customers with lat/lon, popup = "customer_name: monthly_total VND").
p. Compute top10: top 10 customers by sum total_value, top 10 products by sum total_value (using products Map for names).

import { NextRequest, NextResponse } from 'next/server'
import { requireAdmin } from '@/lib/admin/auth'
import { getDashboardData } from '@/lib/admin/services/dashboard'

export async function GET(request: NextRequest) {
  const auth = await requireAdmin()
  if (auth instanceof NextResponse) return auth

  const { searchParams } = new URL(request.url)
  const npp = searchParams.get('npp') || ''
  const month = searchParams.get('month') || new Date().toISOString().slice(0, 7)
  const nganhHang = searchParams.get('nganhHang') || ''
  const thuongHieu = searchParams.get('thuongHieu') || ''
  const kenh = searchParams.get('kenh') || 'le'  // default Kenh le per spec

  try {
    const data = await getDashboardData({ npp, month, nganhHang, thuongHieu, kenh })
    return NextResponse.json(data)
  } catch (error) {
    console.error('Dashboard API error:', error)
    return NextResponse.json({ error: 'Internal server error' }, { status: 500 })
  }
}
