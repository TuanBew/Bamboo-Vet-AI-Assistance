Implementation:
1. `const db = createServiceClient()`

2. **KPIs:**
   - `total_new_sessions`: `SELECT COUNT(DISTINCT c.id) FROM conversations c WHERE EXTRACT(YEAR FROM c.created_at) = $year AND EXTRACT(MONTH FROM c.created_at) = $month`
   - `total_new_queries`: `SELECT COUNT(*) FROM messages m JOIN conversations c ON c.id = m.conversation_id WHERE m.role = 'user' AND EXTRACT(YEAR FROM c.created_at) = $year AND EXTRACT(MONTH FROM c.created_at) = $month`
   - `total_new_users`: `SELECT COUNT(*) FROM profiles WHERE is_admin = false AND EXTRACT(YEAR FROM created_at) = $year AND EXTRACT(MONTH FROM created_at) = $month`
   - `avg_queries_per_session`: `total_new_queries / total_new_sessions` (computed in code, handle division by zero)
   - `total_new_documents`: `SELECT COUNT(*) FROM kb_documents WHERE status = 'active' AND EXTRACT(YEAR FROM created_at) = $year AND EXTRACT(MONTH FROM created_at) = $month`
   - `avg_session_duration_min`: Raw SQL query:
     ```sql
     SELECT COALESCE(
       AVG(
         GREATEST(
           EXTRACT(EPOCH FROM (max_ts - min_ts)) / 60,
           1
         )
       ),
       0
     ) AS avg_session_duration_min
     FROM (
       SELECT c.id, MIN(m.created_at) AS min_ts, MAX(m.created_at) AS max_ts
       FROM conversations c
       JOIN messages m ON m.conversation_id = c.id
       WHERE EXTRACT(YEAR FROM c.created_at) = $year
         AND EXTRACT(MONTH FROM c.created_at) = $month
       GROUP BY c.id
     ) sub
     ```
     Use `db.rpc()` or `db.from().select()` with raw SQL via Supabase. If Supabase JS client doesn't support raw SQL directly, use the postgres REST endpoint or compute in JS from message timestamps.

     **Practical approach:** Query conversations + their min/max message timestamps, then compute average in JavaScript:
     ```typescript
     const { data: sessions } = await db.rpc('get_session_durations', { p_year: year, p_month: month })
     // OR if no RPC, fetch conversations for the month, then for each get min/max message timestamps
     // Simpler: use mv_daily_queries which has session_count per day
     ```
     Actually, since raw SQL may not be directly available via Supabase JS, compute it in JS:
     - Fetch conversations for the month with their messages' created_at timestamps
     - For each conversation, compute duration as (max - min) / 60000 (ms to minutes), minimum 1
     - Average all durations
     - This is acceptable for a single month's data (~200-300 conversations)

3. **daily_query_volume:** Query `mv_daily_queries`: `SELECT day, SUM(query_count)::int as query_count FROM mv_daily_queries WHERE year = $year AND month = $month GROUP BY day ORDER BY day`. Parse numeric results with `Number()`.

4. **daily_sessions:** Same view: `SELECT day, SUM(session_count)::int as session_count FROM mv_daily_queries WHERE year = $year AND month = $month GROUP BY day ORDER BY day`.

5. **recent_sessions (latest 20):** Query conversations for the month, joined with profiles for user_name and messages for query_count and duration:
   ```typescript
   // Fetch recent conversations with user info
   const { data: convos } = await db
     .from('conversations')
     .select('id, created_at, user_id, profiles!inner(full_name)')
     .gte('created_at', `${year}-${String(month).padStart(2,'0')}-01`)
     .lt('created_at', nextMonthStart)
     .order('created_at', { ascending: false })
     .limit(20)
   ```
   For each conversation, compute:
   - `session_code`: first 8 chars of id, uppercased (e.g., "A3F7C219")
   - `date`: formatted created_at
   - `user_name`: from joined profiles
   - `query_count`: count of user messages in this conversation
   - `duration_min`: (max - min message timestamp) / 60, minimum 1

6. **top_questions (top 10):** Query user messages for the month, truncate content to 60 chars, group by prefix, count, return top 10:
   ```typescript
   const { data: messages } = await db
     .from('messages')
     .select('content, conversation_id')
     .eq('role', 'user')
     .in('conversation_id', conversationIdsForMonth)
   // Group by content.slice(0, 60), count, sort desc, take 10
   ```
   Do the grouping in JavaScript since Supabase doesn't support LEFT(content, 60) grouping directly.

7. **category_stats:** Query `mv_category_stats` for the specific year/month. Aggregate by drug_group, animal_type, query_type separately. Sort by count desc.

Parse all numeric values with `Number()` to avoid string issues.

import { NextRequest, NextResponse } from 'next/server'
import { requireAdmin } from '@/lib/admin/auth'
import { getNewActivityData } from '@/lib/admin/services/new-activity'

export async function GET(request: NextRequest) {
  const auth = await requireAdmin()
  if (auth instanceof NextResponse) return auth

  const { searchParams } = new URL(request.url)
  const now = new Date()
  const year = parseInt(searchParams.get('year') || String(now.getFullYear()))
  const month = parseInt(searchParams.get('month') || String(now.getMonth() + 1))

  try {
    const data = await getNewActivityData({ year, month })
    return NextResponse.json(data)
  } catch (error) {
    console.error('New Activity API error:', error)
    return NextResponse.json({ error: 'Internal server error' }, { status: 500 })
  }
}
