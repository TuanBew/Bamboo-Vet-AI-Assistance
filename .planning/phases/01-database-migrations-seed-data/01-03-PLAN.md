**3. Seed functions (in dependency order):**

a) `seedClinics()`:
- Parse `data/seeds/clinics.md`
- For each row, INSERT into clinics with ON CONFLICT (code) DO NOTHING
- Use supabase.from('clinics').upsert() with { onConflict: 'code', ignoreDuplicates: true }
- Return count of inserted rows

b) `seedProfiles()`:
- Parse `data/seeds/profiles.md`
- For each row:
  1. Call `supabase.auth.admin.createUser({ email: row.email, email_confirm: true, password: 'bamboo-seed-2024' })` — wrap in try/catch, log "already exists" on error
  2. If createUser succeeded, the trigger auto-creates a profiles row. Then UPDATE profiles SET full_name, is_admin, clinic_id, province, district, lat, lng, user_type WHERE id = auth_user.id
  3. If createUser failed (user exists), still run the UPDATE to ensure profile fields are set
- Return count of created/updated users

c) `seedConversations()`:
- Parse `data/seeds/conversations.md` for template rows
- Insert template conversations with ON CONFLICT (id) DO NOTHING
- Then PROGRAMMATICALLY GENERATE additional conversations to reach ~4000 total:
  - Read existing non-admin profiles from the database
  - For each month from Jan 2024 to Mar 2026, generate conversations following the volume curve:
    - 2024: ~80/month
    - 2025 H1: ~180/month
    - 2025 H2: ~320/month
    - 2026 Q1: ~280/month
  - Subtract already-inserted template conversations for that month
  - For each generated conversation: random user_id from profiles, random Vietnamese title, random timestamp within that month
  - Use crypto.randomUUID() for ids
  - Insert in batches of 100 with ON CONFLICT (id) DO NOTHING
- Return total count

d) `seedMessages()`:
- For each conversation in the database (query all conversation ids):
  - Check if messages already exist for this conversation (for idempotency)
  - Generate 3-7 messages (avg 5) alternating user/assistant roles
  - Vietnamese content: user messages are veterinary questions, assistant messages are answers
  - Use short templates (2-3 sentences each) — these are seed data, not real AI responses
  - Insert with ON CONFLICT (id) DO NOTHING
- Return count

e) `seedQueryEvents()`:
- Parse `data/seeds/query_events.md` for template rows
- Insert template query_events with ON CONFLICT (conversation_id) DO NOTHING
- Then for each programmatically-generated conversation (those without a query_event):
  - Generate a query_event with:
    - Same user_id and created_at as the conversation
    - clinic_id from the user's profile
    - Random drug_category following distribution: khang_sinh 35%, vitamin 20%, vac_xin 18%, hormone 12%, khang_ky_sinh_trung 10%, khac 5%
    - Random animal_type following distribution: trau_bo 30%, lon 25%, ga 20%, cho_meo 15%, thuy_san 7%, khac 3%
    - Random query_type following distribution: dieu_tri 35%, chan_doan 28%, lieu_luong 20%, phong_benh 12%, khac 5%
    - Random response_time_ms between 800-5000
  - Insert in batches with ON CONFLICT (conversation_id) DO NOTHING
- Return count

f) `seedKbDocuments()`:
- Parse `data/seeds/kb_documents.md`
- For each row, INSERT into kb_documents with ON CONFLICT (doc_code) DO NOTHING
- Note: do NOT include `id` in the insert — let the database generate it
- Return count

**4. Main function:**
```typescript
async function main() {
  console.log('=== Bamboo Vet Seed Script ===\n')

  // Check if already seeded
  const { count } = await supabase.from('clinics').select('*', { count: 'exact', head: true })
  if (count && count > 0) {
    console.log(`Database already has ${count} clinics. Running in idempotent mode (will skip existing records).\n`)
  }

  const clinics = await seedClinics()
  const profiles = await seedProfiles()
  const conversations = await seedConversations()
  const messages = await seedMessages()
  const queryEvents = await seedQueryEvents()
  const kbDocs = await seedKbDocuments()

  console.log('\n=== Seed Complete ===')
  console.log(`Clinics:       ${clinics.inserted} inserted, ${clinics.skipped} skipped`)
  console.log(`Profiles:      ${profiles.inserted} inserted, ${profiles.skipped} skipped`)
  console.log(`Conversations: ${conversations.inserted} inserted, ${conversations.skipped} skipped`)
  console.log(`Messages:      ${messages.inserted} inserted, ${messages.skipped} skipped`)
  console.log(`Query Events:  ${queryEvents.inserted} inserted, ${queryEvents.skipped} skipped`)
  console.log(`KB Documents:  ${kbDocs.inserted} inserted, ${kbDocs.skipped} skipped`)
}
```
