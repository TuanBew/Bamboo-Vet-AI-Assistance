### Task 2: Generate and insert inventory snapshots (daily, 2024-2026)

Generate daily inventory snapshots for every product from Jan 1, 2024 to Mar 29, 2026.

**Data generation rules:**
- Base quantity per product: random integer between 50 and 500 units at start of 2024
- Daily fluctuation: ±0–15% random change (mostly small, occasional large drops for "sales")
- Seasonal patterns:
  - Q1 (Jan-Mar): High replenishment → quantities trend UP (farmers preparing for year)
  - Q2 (Apr-Jun): Moderate sales → stable
  - Q3 (Jul-Sep): Peak usage season → quantities trend DOWN (livestock disease season)
  - Q4 (Oct-Dec): End-of-year push → high sales, then replenishment in Dec
- Products don't go below 5 units (min floor)
- Products don't exceed 1000 units (warehouse capacity)
- Unit price: use the price from products table, add ±5% quarterly variation

**Implementation:**
- Write a TypeScript script `scripts/seed-sales.ts`
- Use `createClient(url, serviceRoleKey)` from `@supabase/supabase-js`
- Load credentials from `.env.local` using `dotenv`
- Fetch all product IDs from DB first
- Generate snapshots in batches of 1000 rows (Supabase insert limit)
- Use `ON CONFLICT (product_id, snapshot_date) DO NOTHING` (idempotent)
- Print progress every 5000 rows

**Volume check:** 65 products × ~820 days = ~53,300 rows. Batch insert in chunks of 500.

Commit: `feat(01-04): generate daily inventory snapshots Jan 2024 – Mar 2026`
