# Quick Task 260404-c75: Ton Kho Inventory Panel with Real Data Summary

Rewrote Ton Kho page to query real inloc + product tables with paginated fetching, JS-side latest-date deduplication, and dynamic NPP/brand filter dropdowns.

## Task Results

| # | Task | Commit | Key Changes |
|---|------|--------|-------------|
| 1 | Rewrite ton-kho service + API route | a3926c54 | Full service rewrite querying inloc/product tables, API accepts npp+brand params, page/loader pass filters, VI keys added |
| 2 | Update TonKhoClient UI | 7c8086e3 | NPP dropdown dynamic, brand dropdown replaces nhom, DataTable columns updated, filter state uses npp+brand |

## What Changed

### Service Layer (lib/admin/services/ton-kho.ts)
- **Before**: Queried non-existent `inventory_snapshots`, `products`, `purchase_orders`, `purchase_order_items` tables
- **After**: Queries real `inloc` table (paginated with .range()) and `product` table
- Deduplication: finds MAX(inv_date) per (site_code, sku_code), then SUMs onhand_qty across warehouses
- Value calculation: qty * last_cost (from product table join)
- New `filter_options` field provides dynamic NPP list and brand list for dropdowns
- New `TonKhoFilters` interface: snapshot_date, npp, brand, search (replaces old nhom field)

### API Route (app/api/admin/ton-kho/route.ts)
- Added `npp` and `brand` query parameter extraction
- Removed old `nhom` parameter

### Client Component (app/admin/ton-kho/TonKhoClient.tsx)
- NPP dropdown: populated from filter_options.npps (3 NPPs: BAMBOOVET, MEGAVET MIEN BAC, MEGAVET MIEN NAM)
- Brand dropdown: replaces old nhom dropdown, populated from filter_options.brands
- DataTable columns: sku_code, sku_name, qty, product (nganh hang), brand, unit_price (gia chia), total_value
- Removed: min_stock, last_import_date columns (not available in real data)
- Initial filter_options stored in useMemo to prevent cascade-empty dropdowns

### VI Keys (lib/i18n/vietnamese.ts)
- Added: allBrands, brand, nganh, giaChia to tonKho section
