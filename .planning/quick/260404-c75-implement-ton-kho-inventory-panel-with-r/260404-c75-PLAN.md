3. Implement `getTonKhoData(filters)`:
   - **Step A — Get filter options** (always, regardless of filters): Query `inloc` for DISTINCT site_code, site_name pairs. Query `inloc` for DISTINCT brand values (non-null, non-empty). These populate the filter dropdowns.
   - **Step B — Get latest inventory date per (site_code, sku_code)**: Query `inloc` with `.lte('inv_date', filters.snapshot_date)`. If `filters.npp` is non-empty, add `.eq('site_code', filters.npp)`. Select `site_code, sku_code, inv_date`. Use JS to compute MAX(inv_date) per (site_code, sku_code) — this handles each NPP having different latest dates. NOTE: Supabase JS cannot do GROUP BY, so fetch all rows and deduplicate in JS using a Map keyed by `${site_code}|${sku_code}`.
   - **Step C — Fetch inventory rows for those latest dates**: For each (site_code, sku_code, latest_date) combo, we need the actual rows to SUM onhand_qty across warehouses. In practice, re-query `inloc` with the same date filter and do the warehouse dedup in JS: build a Map keyed by `${site_code}|${sku_code}` that SUMs onhand_qty across all whse_code rows for the latest inv_date.
   - **OPTIMIZATION**: Steps B and C can be combined — fetch all inloc rows with `.lte('inv_date', snapshot_date)` (and optional `.eq('site_code', npp)`), then in JS: (1) find latest inv_date per (site_code, sku_code), (2) filter to only rows matching that latest date, (3) SUM onhand_qty per (site_code, sku_code). This is ONE query instead of two.
   - **IMPORTANT**: The inloc table may have many rows. Use `.select('site_code, site_name, sku_code, sku_name, inv_date, onhand_qty, category, brand, product')` to limit columns. If data is too large, consider pagination with `.range()` in a loop. For MVP, a single query should work since there are only 3 NPPs.
   - **Step D — Join with product table**: Query `product` table `.select('site_code, sku_code, last_cost')`. Build a Map keyed by `${site_code}|${sku_code}` -> last_cost. Join with inventory data to get unit_price.
   - **Step E — Apply brand and search filters in JS**: Filter the joined data by `filters.brand` (match on brand column) and `filters.search` (ILIKE on sku_name or sku_code).
   - **Step F — Compute KPIs**: total_value = SUM(qty * unit_price), total_qty = SUM(qty), sku_in_stock = COUNT where qty > 0, total_sku = total count.
   - **Step G — Compute chart aggregations**: Group by category (nhom), brand (thuong hieu), product (nganh hang) for both value and qty. Use the same groupBy helper pattern as current code.
   - **Step H — Build products array**: Map each item to {sku_code, sku_name, qty, product, brand, unit_price, total_value}. Sort by total_value DESC.
